### 3.2 Hash Computation
```kotlin
suspend fun computeFileHash(file: File): String = withContext(Dispatchers.IO) {
    val md = MessageDigest.getInstance("SHA-256")
    file.inputStream().use { `in` ->
        val buffer = ByteArray(8192)
        var read: Int
        while (true) {
            read = `in`.read(buffer)
            if (read == -1) break
            md.update(buffer, 0, read)
        }
    }
    Base64.encodeToString(md.digest(), Base64.NO_WRAP)
}
```

### 3.4 Execution Flow
```
1. Initialize: Get/create manifest message in Saved Messages
2. For each BackupFolder:
   a. Scan files recursively
   b. For each file: compute hash + modified time
   c. Check manifest for existing entry by path
   d. If missing OR hash differs: upload with caption "backup/{folder}/{name}"
   e. Update in-memory manifest
   f. Add 1.5s delay after each upload
3. Save updated manifest to Telegram (edit pinned message)
4. Report final stats via notification
```

## Key Technical Decisions

### Manifest Storage
- Single pinned message in Saved Messages (folderId = null)
- JSON format for easy parsing
- Edit operation to update (efficient)

### Deduplication
- SHA-256 hash as primary identifier
- File path as key for organization
- Handles file moves by hash (same content, new location)

### Rate Limiting
- 1.5s delay between uploads
- Prevents Telegram throttling

### Error Handling
- Continue on individual file failures
- Manifest persistence enables resume
- Final report shows stats
